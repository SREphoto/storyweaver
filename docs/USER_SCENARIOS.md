# StoryWeaver User Scenarios & Workflows

This document outlines 20 unique user personas and their typical workflows within StoryWeaver. These scenarios demonstrate the app's versatility across different creative needs.

---

## 1. The Pantser (Discovery Writer)

* **Goal**: Start writing without a plan and let the story evolve.
* **Workflow**:
    1. Opens **Story Editor**.
    2. Starts typing directly in the `Active Output` or a new Scene.
    3. Stuck? Uses **AI Assistant** -> "Give me a plot twist for this scene."
    4. Uses **Character Builder** *during* writing to define a character they just invented.
    5. **Visuals**: Generates character art only after the character has "spoken" to them.

## 2. The Architect (Planner)

* **Goal**: Meticulously plan every detail before writing prose.
* **Workflow**:
    1. Starts in **Timeline View**. Creates the entire beat sheet of the story.
    2. Moves to **Character Builder**. Defines all 5 main characters with full profiles.
    3. Goes to **Story Map**. Generates the world map and pins key locations.
    4. Finally, creates empty **Scenes** for each timeline event.
    5. Uses **Screenplay Mode** to draft dialogue before narrative.

## 3. The Visual Novelist

* **Goal**: Create a graphical text adventure.
* **Workflow**:
    1. Focuses heavily on **Character Builder** and **Asset Library**.
    2. Generates multiple expressions for each character (Happy, Sad, Angry).
    3. Uses **Visual Inspiration** to create mood backgrounds for every scene.
    4. Writes in **Visual Novel View** (if available) or standard Scenes, attaching specific images to each block of text.
    5. Exports assets to a game engine (e.g., Ren'Py).

## 4. The Dungeon Master (DM)

* **Goal**: Build a campaign setting for a TTRPG.
* **Workflow**:
    1. Lives in **Story Map**. Generates the world, adds pins for towns/dungeons.
    2. Uses **World Builder** (Lore) to define magic systems and politics.
    3. Uses **Character Builder** for NPCs (Non-Player Characters).
    4. Uses **Object Generator** to create legendary items and loot.
    5. During the session, uses **Quick Notes** to track player actions.

## 5. The Screenwriter

* **Goal**: Write a movie or TV show pilot.
* **Workflow**:
    1. Selects **Screenplay Mode** in Scene Editor.
    2. Uses **Storyboard Modal** to visualize key shots.
    3. Focuses on **Dialogue Generator** to refine banter.
    4. Uses **Video Analysis** to break down reference clips from generic movies for pacing.
    5. Exports specifically in standard script format.

## 6. The Comic Book Creator

* **Goal**: Visual storytelling.
* **Workflow**:
    1. Uses **Comic Creator** tool primarily.
    2. Defines "Hero" and "Villain" visuals strictly to ensure consistency.
    3. Generates panels beat-by-beat.
    4. Uses "Regenerate Region" (Inpainting) to fix hands/faces in panels.
    5. Adds speech bubbles in an external tool (or StoryWeaver if supported).

## 7. The World Builder (Lore Junkie)

* **Goal**: Create a universe (Encyclopedia Galactica).
* **Workflow**:
    1. Ignores the Story Editor initially.
    2. Spends hours in **Story Map** and **Asset Library**.
    3. Generates detailed histories for nations, religions, and technologies.
    4. Uses **Relationship Web** to map out political alliances.
    5. Eventually writes "Historical Documents" as scenes.

## 8. The Fan Fiction Writer

* **Goal**: Write stories in an existing universe.
* **Workflow**:
    1. Imports existing characters into **Character Builder** (manually or via analysis).
    2. Uses **Video Analysis** on clips from the source material to capture the "vibe".
    3. Writes mostly in **Story Editor**.
    4. Uses **AI Assistant** to check "Is this in character for [Name]?"

## 9. The Roleplayer (Text-based RP)

* **Goal**: collaborate on a story or develop a specific OC (Original Character).
* **Workflow**:
    1. Deep dives into **Character Builder** for a single character (Theirs).
    2. Generates infinite outfits and scenarios for that one character.
    3. Uses **Dialogue Generator** to simulate responses.
    4. Keeps a **Timeline** of the RP session.

## 10. The Educational Storyteller (Teacher/Parent)

* **Goal**: Create a personalized bedtime story or lesson.
* **Workflow**:
    1. Inputs child's name and favorite things into **Story Context**.
    2. Uses **Quick Generation** for a "One-Shot" story.
    3. Generates distinct, friendly visuals for each page.
    4. Reads directly from **Book View** (Clean Mode).

## 11. The Mystery Writer

* **Goal**: A complex whodunit.
* **Workflow**:
    1. Heavily uses **Relationship Web** to track suspects and motives.
    2. Uses **Timeline View** to manage two timelines: "The Crime" and "The Investigation".
    3. Uses **Object Generator** for clues (The Bloody Knife, The Letter).
    4. Writes scenes out of order to plant foreshadowing.

## 12. The Horror Writer

* **Goal**: Scares and atmosphere.
* **Workflow**:
    1. Uses **Visual Inspiration** with specific "Dark/Eerie" prompts to set the mood.
    2. Writes with a focus on **Sensory Details** (Sound/Smell).
    3. Uses **AI Assistant** to "Make this description more unsettling."
    4. Avoids clear character visuals to keep the monster ambiguous.

## 13. The NaNoWriMo Sprinter

* **Goal**: Word count above all else.
* **Workflow**:
    1. Uses **Distraction Free Mode** (Full Screen).
    2. Uses **Sprint Timer** (if available) or external timer.
    3. Uses **AI Assistant** -> "Continue" whenever they hit writer's block to keep momentum.
    4. Organizes nothing until November is over.

## 14. The Graphic Novel Adapter

* **Goal**: Turn a classic book into a graphic novel.
* **Workflow**:
    1. Pastes original text into **Scene Editor**.
    2. Uses **Summarizer** to condense prose into captions/dialogue.
    3. Uses **Storyboard Tool** to break text into panels.
    4. Generates reference images for artists.

## 15. The Game Dev (Indie)

* **Goal**: Lorem and assets for a game.
* **Workflow**:
    1. Uses **Asset Library** as a repository.
    2. Generates texture ideas and item icons using **Object Generator**.
    3. Uses **Story Map** for level design inspiration.
    4. Exports JSON data from **World View**.

## 16. The Collaborative Duo

* **Goal**: Two writers, one story.
* **Workflow**:
    1. Heavily rely on **Notes/Comments** (if available) or distinct Character profiles.
    2. One focuses on **World/Map**, the other on **Dialogue**.
    3. Use **Timeline** to sync their separate plot threads.

## 17. The Historical Fiction Writer

* **Goal**: Accuracy and period feel.
* **Workflow**:
    1. Uses **AI Assistant** to research: "What did 18th-century French peasants eat?"
    2. Generates visuals strictly with period-accurate prompts.
    3. Uses **Timeline** to align story events with real history.

## 18. The Dreamer (Idea Collector)

* **Goal**: Catalog ideas, maybe write later.
* **Workflow**:
    1. Uses **Quick Generate** for 50 different "Plot Ideas".
    2. Saves the best ones to **Asset Library**.
    3. Generates cool character portraits but no bio.
    4. Never actually writes a scene (and that's okay!).

## 19. The Editor / Polisher

* **Goal**: Refine an existing draft.
* **Workflow**:
    1. Imports a full draft.
    2. Selects scene -> **AI Assistant**: "Critique the pacing."
    3. Selects paragraph -> **AI Assistant**: "Show, don't tell."
    4. Uses **Timeline** to check for plot holes.

## 20. The Marketing Guru (Book Launch)

* **Goal**: Create promotional material for the story.
* **Workflow**:
    1. Takes the finished character -> **Video Creator** -> "Cinematic trailer teaser".
    2. Generates "Cover Art" ideas in **Asset Library**.
    3. Uses **Summary Tool** to write the back-of-book blurb.
    4. Creates social media content using character quotes and images.
